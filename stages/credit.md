# Credit

## 目标

实现一个程序，验证信用卡号的有效性并识别卡类型。

## 背景

信用卡号使用 **Luhn 算法**（又称 mod 10 算法）进行校验。

### Luhn 算法

1. 从右向左，从倒数第二位开始，每隔一位乘以 2
2. 将所有乘积的各位数字相加（例如 12 变成 1 + 2 = 3）
3. 将未乘以 2 的数字也加到总和中
4. 如果总和的最后一位是 0，则卡号有效

### 卡类型识别

- **AMEX**: 以 34 或 37 开头，15 位
- **MASTERCARD**: 以 51-55 开头，16 位
- **VISA**: 以 4 开头，13 或 16 位

## 规范

- 程序文件名必须是 `credit.c`
- 提示用户输入卡号
- 输出 `AMEX`、`MASTERCARD`、`VISA` 或 `INVALID`

## 示例

```
$ ./credit
Number: 378282246310005
AMEX
```

```
$ ./credit
Number: 4111111111111111
VISA
```

```
$ ./credit
Number: 1234567890
INVALID
```

## 提示

1. 使用 `get_long` 获取卡号（不要用 `get_string`）
2. 使用 `%` 和 `/` 运算符提取各位数字
3. 先实现 Luhn 校验，再实现类型识别
